Tabelas necessárias para o projeto, contando com quais colunas cada uma terá que ter:


1. Tabela de Cadastro de Cliente:

1.1 Nome do Cliente - nome_cliente 
1.2 CPF ou CNPJ do Cliente - cpf_cliente (PK)
1.3 Telefone do Cliente - telefone_cliente
1.4 Email do Cliente - email_cliente
1.5 Endereço do Cliente - endereco_cliente
1.6 Modelo do carro e ano do mesmo - carro_cliente
1.7 Data de nascimento do Cliente - data_cliente

Explicação: 

- Cliente pediu uma tabela de cadastro que contenha os dados do cliente. 
Esses dados deveriam ser o nome do cliente, o CPF dele, o telefone para contato, o Email para caso precise enviar algo pelo Email, endereço para auxiliar numa possível entrega, o modelo do carro e o ano dele (para melhor compreedimento do que será necessário) e a data de nascimento do cliente.

2. Tabela de Produtos:

2.1 Nome do produto - nome_produto 
2.2 ID do produto - id_produto
2.3 Quantidade do produto - quantidade_produto
2.4 Código de barras do produto - cod_produto (PK)
2.5 Preço do produto - preco_produto
2.6 Status do produto - status_produto
2.7 Observação do produto - obs_produto

- Cliente pediu uma tabela de produtos para um cadastro e para conferir futuramente o estoque.
Terá que ter o nome do produto, um id de cadastro (em ordem de qual foi cadastrado primeiro), quantidade do produto, o código de barras do produto, o preço do mesmo, o status indicando se ele está (ativo, inativo, fora de estoque) e por fim uma observação caso necessária.

3. Tabela de Vendas:

3.1 ID da venda - id_venda (PK)
3.2 Cliente que comprou - cpf_cliente (Chave estrangeira)
3.3 Data da venda - data_venda
3.4 Item comprado - cod_produto (Chave estrangeira)
3.5 Quantidade comprada - quantidade_produto_vendido
3.6 Valor total da venda - valor_total_venda
3.7 Forma de pagamento - pagamento_venda
3.8 Status da venda - status_venda

- Cliente pediu uma tabela com as informações da venda, possibilitando depois uma impressão do comprovante que apareçam os dados pedidos.
O id da venda (informando o número da venda, em ordem de qual venda foi feita primeira), o cliente que comprou (partindo do CPF), a data da venda, o item comprado (código de barras, pegando os dados do produto), quantidade comprada do mesmo, o valor total da venda, a forma de pagamento da venda (cartão, dinheiro, pix) e o status da venda (Paga, pendente ou deletada)

4. Tabela de Pedidos(Fornecedores):

4.1 ID da nota - id_nota (PK)
4.2 CNPJ do Fornecedor - cnpj_fornecedor
4.3 Data da nota - data_nota
4.4 Valor da nota - valor_nota
4.5 Status da nota - status_nota
4.6 Quantidade do produto pedido - quantidade_produto_pedido
4.7 Nome do produto pedido - nome_produto_pedido

- Cliente pediu uma tabela para deixar armazenado os pedidos feitos à fornecedores.
O id da nota (informando o número da nota, em ordem de qual pedido foi feito primeiro), o fornecedor, a data da nota, o valor da nota, o status da nota (Paga, pendente, deletada), o nome do produto pedido e a quantidade pedida

5. Tabela de Fornecedores:

5.1 ID do Fornecedor -id_fornecedor 
5.2 Nome do Fornecedor - nome_fornecedor
5.3 CNPJ do Fornecedor - CNPJ - cnpj_fornecedor (PK)
5.4 Telefone do Fornecedor - telefone_fornecedor
5.5 Endereço do Fornecedor - endereco_fornecedor

- Cliente pediu uma tabela para deixar armazenado dados dos fornecedores.
O id do fornecedor, o nome do fornecedor, o CNPJ do fornecedor, o telefone e o endereço da empresa.
